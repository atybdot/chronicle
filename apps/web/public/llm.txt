# Chronicle

> AI-powered CLI tool that transforms uncommitted changes into atomic, well-organized git commits with realistic timestamps. Uses LLMs to analyze code changes, generate meaningful commit messages, and backfill git history with realistic commit patterns.

## Overview

Chronicle is a developer tool that helps maintain clean git history by:

- Analyzing uncommitted changes and generating intelligent commit plans
- Splitting large changes into atomic commits with realistic timestamps
- Supporting multiple LLM providers (OpenAI, Anthropic, Google Gemini, OpenRouter, Ollama)
- Distributing commits realistically across time with work hours and weekend handling

## Installation

```bash
bun i -g @atybdot/chronicle
```

## Commands

### analyze

Analyze uncommitted changes and generate a commit plan.

- `--path` - Path to git repository
- `--date-range` - Date range for commit distribution
- `--include-staged` - Include staged changes
- `--include-unstaged` - Include unstaged changes
- `--include-untracked` - Include untracked files

### backfill

Backfill git history with intelligent commit splitting.

- `--path` - Path to git repository
- `--date-range` - Date range (e.g., 'spread over 2 weeks')
- `--start-date` - Start date (ISO format)
- `--end-date` - End date (ISO format)
- `--dry-run` - Preview without making changes
- `--interactive` - Interactive mode with prompts
- `--output` - Output format: visual, json, minimal

### status

Show status of current repository changes.

- `--path` - Path to git repository

### config

Configure chronicle settings.

- `init` - Interactive setup wizard
- `show` - Show current configuration
- `set` - Set a configuration value
- `get` - Get a configuration value
- `model` - Change the LLM model
- `provider` - Change the LLM provider
- `prompt` - Set custom AI instructions
- `clear-cache` - Clear the AI response cache
- `cache-stats` - Show cache statistics
- `telemetry` - Manage telemetry settings

## Configuration

Configure LLM provider and model, git author details, and commit distribution settings.

### Supported Providers

- `openai` - OpenAI (GPT-4o, GPT-4o mini, etc.)
- `anthropic` - Anthropic (Claude)
- `gemini` - Google Gemini
- `openrouter` - OpenRouter (aggregates many models)
- `ollama` - Ollama (local models, no API key required)

### Configuration Options

- `llm.provider` - LLM provider (default: openai)
- `llm.model` - Model name (default: provider default)
- `llm.apiKey` - API key (from environment: CHRONICLE_AI_KEY)
- `llm.baseUrl` - Custom endpoint for API-compatible providers
- `llm.customPrompt` - Custom AI instructions
- `git.authorName` - Git author name (default: from git config)
- `git.authorEmail` - Git author email (default: from git config)
- `defaults.distribution` - Commit distribution (realistic, linear, concentrated)
- `defaults.dryRun` - Preview by default (default: true)
- `defaults.workHoursStart` - Work hours start (default: 9)
- `defaults.workHoursEnd` - Work hours end (default: 18)
- `defaults.excludeWeekends` - Skip weekends (default: false)
